## Solution/Approach
- Web component wraps React AtlasPanel; light DOM (not shadow) so app.css + :root --atlas-* theme apply.
- vite.embed.config.js: build.lib IIFE, inlineDynamicImports:true, vite-plugin-css-injected-by-js@^3.5.2 (v5 wanted vite8; pinned v3). npm run build:embed.
- embed-shim.js imported FIRST (ESM eval order) sets globalThis.process before react-dom evaluates (CDN drop-in has no process global). Also vite define process.env.NODE_ENV.
- d2 WASM: loadD2() already does import(/* @vite-ignore */ 'https://esm.sh/@terrastruct/d2') → stays external/CDN. ensureD2() memoized, file:// guard, probe-compile('a -> b') inside a 4s Promise.race timeout (catches import-ok-but-wasm-hangs).

## Key Insights
- BUG FIXED: miniParseD2 didn't expand chains — `a -> b -> c` made a node literally named "b -> c" (lazy regex). Now splits on ->, trailing ": label" → last hop. (src/core/d2.js)
- BUG FIXED: computeHeat (graphology betweenness) threw "mnemonist/fixed-stack capacity should be positive" on some graphs and BLANKED the whole render — wrapped try/catch + order===0 guard.
- BUG FIXED (the big one): custom-element load-order. <script src> in <head> defines the element BEFORE the parser reaches <atlas-graph>, so connectedCallback fires before the inner config <script> child exists → empty config → blank graph. Fix: if document.readyState==='loading', wait DOMContentLoaded then mount. Worked before only because first test had the script at end of <body>.
- d2 wasm hang on file:// is in compile() (wasm binary fetch), NOT the import (import resolves). Timeout must wrap a probe-compile, not just the load. And file:// can never run it → skip entirely.
- jsDelivr-over-gh serves committed repo files (no build on their side) → must force-add the gitignored dist/atlas.js. @main caches ~12h; purge via purge.jsdelivr.net/... ; pin @<sha> for immutable.
- Architecture = hourglass: many source dialects → buildModel → MODEL {entities,edges,refs} (narrow waist) → AtlasPanel → two shells (deck Frames.jsx / <atlas-graph>) → two ship lanes (vite build / CDN).
