## Problem/Context
Resumed from anim-core-ts-refactor-tours-multiselect session (Rep 1 done). Rep 2 had two
open calls: where the span surface renders, and tour authoring syntax.

## Solution/Approach
Decisions made: (1) spotlight lives inside AtlasPanel as an overlay over .atlas-graph --
the tour player lives there and span steps leave the graph view unchanged; the component is
standalone so Frames can mount it later. (2) Tour authoring reuses the rel target encoding
verbatim via `# tour`, so a .dl-emitted tour_step row and a hand-written annotation are the
same string -- answers the `# step` vs `# tour` open question.

## Key Insights
- One string codec for targets everywhere: parseTarget in codec.ts serves rel rows, ?av=
  focus sets, and `# tour` lines. Authoring and db emission converge by construction.
- The once-bound window.__atlas closure goes stale against React state: mirror mutable
  position/spot in S.current and call through a ref (stepTourRef).
- vite build empties dist/, so npm run build:embed must run AFTER npm run build or
  dist/atlas.js disappears (emptyOutDir:false only on the embed config).
- Adding a frame mid-deck shifts every later frame index: e2e frame constants are
  positional and must be re-verified against frames.json after deck edits.
- Annotation regex needs whitespace-before-colon (`\s+:`) to split comment from target,
  since span targets contain ':' (file:lo..hi) and \S+ is greedy.
- Band geometry is pure (core/spotlight.ts); the renderer owns only DOM, the transition
  CSS, and smooth scroll. LINE_H constant pinned to CSS line-height by comment on both ends.
