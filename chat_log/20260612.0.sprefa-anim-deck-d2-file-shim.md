## Problem/Context
- d2.js ALWAYS routes compile through a Blob-URL module worker; Chromium refuses worker creation on file:// pages. Inlining wasm (it already ships inline base64) cannot fix that. Confirmed: same bytes OK over http, worker error on file://.
- Mini parser had been silently masking that the template's netstack atlas block was INVALID d2: container id `link` is a reserved keyword. Renamed to l2. Real d2 surfaces errors; mini never could.
- Vite dev cold start: first import('@terrastruct/d2') triggers dep re-optimize + page reload, which made the first shoot pass capture empty canvases. Warm cache fine.

## Solution/Approach
Worker shim (file:// only, idempotent): remember Blobs handed to createObjectURL; fake Worker evaluates blob text via new Function('self', body) after stripping export keywords; self = Proxy whose reads fall through to globalThis with lowercase natives bound (avoids Illegal invocation), writes stay local; postMessage queues until boot.

- esm.sh CDN loading was replaced by bundling; embed inlineDynamicImports folds the 8MB chunk automatically.
