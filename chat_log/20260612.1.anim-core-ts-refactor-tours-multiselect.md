## Problem/Context
User outpoured the whole vision across many turns: codebase learning decks without LLM token
spend, datalog as snapshot state with multi-tag clustering and tours (per-scenario edge
commentary), cards rendered IN the graph, CSS-anchor node editors, code-range spotlight tours
(explain a giant PR / AI session via span-anchored steps), find-refs periscope (hover type ->
fs-tree slice + staggered edge draw-on), egui native consumer, doc daemon. All of it reduced to:
keyed graph states + tours, rendered as animated diffs, sqlite rel_* tables as the contract
between sprefa (producer) and anim (renderer). User then asked to plan/build the pure core.

## Solution/Approach
Audit found core already had pure versions (cone, hopDistances, transition.js) that AtlasPanel
reimplemented imperatively, plus three coexisting step/tour mechanisms. Refactor: extract all
pure logic to core (.ts), unify sequencing into Tour {target: focus[]|reveal[]|path|span,
comment}, route visibility through the diff primitive, declare DESIGN.md interfaces (span
targets, rows loader) with tests even where renderer support lands later. Cone stays a lighting
decoration over the visible set (faded context preserved); only the round player changes the
visible set.

## Key Insights
- Everything in the outpour is one primitive: animated diff between keyed graph states; tours
  are rows (tour_step(tour, seq, target, comment)); a "scenario" is a tour, an "audience" is a
  tag namespace.
- The shoehorn was never datalog; it was gen-splicing rows through markdown comments. sqlite
  rel_* tables are the contract; anim already reads them at render time (sql-graph fences).
- Naming: Target, never Subject (RxJS owns Subject). Multi-select is the base case (NodeId[]).
  Both saved to memory (feedback_anim_naming_and_ts.md).
- Round-player semantics pinned by test: ids never revealed by ANY step are permanent context;
  revealed ids appear cumulatively; edges need both endpoints visible.
- atlas/core is a symlink to src/core: vitest needs an explicit include or suites run twice.
- Canvas classes are not DOM-queryable: e2e drives the panel via window.__atlas hook.
- Cone = decoration, round player = visibility; conflating them through one View would have
  made every focus click an isolate.
